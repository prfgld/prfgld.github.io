Emergency Hard Fork

by BlakeKeiller, Saturday, June 13, 2020, 14:50 (540 days ago)
edited by BlakeKeiller, Saturday, June 13, 2020, 15:32

Version 0.1.2 of the software is now ready. It is an emergency hard fork in response to the massive number of blocks (about 100) yesterday:

https://proofgold.org/proofgold-0.1.2.tgz

Please download it before restarting staking. I have staked one new block to test it, and will continue burning/staking in a few hours from now.

Technically this may be a soft fork, but I cannot guarantee this. The treatment of pure burns is the same as before. The notion of coinage has been modified for stakes so that the number of atoms (plus 1 million times the number of litoshis burned) is multiplied by 0.0001 of the age (in blocks) of the staked asset. If the staked asset is locked, the factor is 0.0002, with a maximum factor of 1. This should make it so that it is more difficult to stake with an asset at first, but very gradually becomes easier. The reason it should be a soft fork is that it should be strictly more difficult to stake now (and never easier) so that valid blocks now should still look valid to old nodes. Nevertheless, it is safer if everyone upgrades.
